Technical Details

This page summarizes the technical solutions that are used to produce this application.

Front-end

  • Angular 7 is used as the base framework
  • Angular Material is used for web components that provide Googles' Material Design-look to the application

Back-end

  • The backend consists of a REST API built with .NET Core
  • The backend is created using the 'Spa'-template: it does not only host the backend API, but also serves the static files that represent the front-end code.

Build & Release

  • GitHub is used to source repository. You can find the code here.
  • Azure Devops is used to implement the Build and Release pipelines
  • The build pipeline builds the code, tests the frontend code (jasmine), tests the backend code (unittests) and published containers.
  • The result of the build pipeline is (as set of) Docker container(s) that are published to a private container repository (Azure Container Repository)
  • The release pipleline published the containers to a Kubernetes cluster using deployment definitions that are published as build artifacts.

Hosting environment

  • The application is hosted in a Kubernetes cluster (Azure Kubernetes Service (AKS))
  • The site and APIs are exposed using both an nginx ingress controller and an istio ingress gateway
  • Istio is deployed in the Kubernetes cluster for live monitoring of the application
  • SSL-certificates (LetsEncrypt) are automatically obtained and refreshed using the cert-manager add-on for Kubernetes
